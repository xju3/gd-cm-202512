# 光模块、尾纤、传输故障	

## 提前准备	
1. 故障定位与告警分析
告警接收：通过“网管监控系统获取 “BBU 通信中断”“网元脱管” 告警”（含站点名称、BBU 网元 ID、所属基站、告警发生时间），同步关联 “光模块告警”“传输链路中断” 等衍生告警（如 “光口接收功率过低”“PTN 链路断”）。
初步判断：基于告警关联关系区分故障方向 ——
光模块 / 尾纤故障：通常伴随 “光口告警”“收光功率异常”（如≤-30dBm），且仅单 BBU 脱管；
传输故障：可能涉及多 BBU 脱管（同传输链路下的站点），伴随 “PTN/OTN 设备告警”“链路误码率过高”。
站点属性确认：通过综资系统明确 ——
移动自有站点：BBU 及配套光模块、尾纤、传输设备（如接入层 PTN）均由移动维护；
铁塔站点：BBU 及光模块、尾纤属移动维护，传输机房设备（如汇聚层 PTN）可能涉及铁塔或传输专业协同。
2. 备件与工具准备
（1）光模块故障：同型号光模块（如 10G SFP+，LC 接口，波长 1310nm），光功率计（量程 - 70~+10dBm，精度 ±0.5dB）、光模块专用拔插工具、防静电手环（接地电阻≤1MΩ）
（2）尾纤故障：同类型尾纤（如 LC-LC 单模，长度 3m/5m，弯曲半径≥30mm），OTDR（光时域反射仪，测试范围 0~100km）、光纤清洁套装（酒精棉、清洁笔）、熔接机（备用，如需现场熔接）
（3）传输故障：传输端口光模块、备用尾纤，传输测试仪（测误码率、吞吐量）、网线（Console 口配置用）、光衰减器（调节光功率）
（4）通用工具：绝缘柄螺丝刀、标签机（标注尾纤对应端口）、手电筒、记录本、告警指示灯对照表（BBU / 传输设备）。
3. 安全与协同准备
（1）资质要求：操作人员需熟悉 BBU 与传输链路拓扑（如 “BBU→光模块→尾纤→PTN 设备” 连接关系），涉及电源操作时持低压电工证；
（2）安全防护：配备安全帽、反光衣、防静电手环（接触光模块 / BBU 时必须佩戴），高空作业（如尾纤布放于铁塔）需持登高证及双钩安全带；
（3）协同机制：若涉及传输专业，提前通过 “传输协同群” 通知传输代维，共享链路拓扑图（含端口编号、光功率基线值）。"
        
        
## 现场操作	

1. 共性进站与状态确认
抵达现场后，签署《基站出入登记单》，明确 BBU 安装位置（机房内机架 / 室外柜）及传输链路接口（BBU 光口→尾纤→传输设备端口）；
现场检查 BBU 状态：观察 RUN 灯（正常 1s 闪烁，脱管时常亮 / 灭）、ALM 灯（红灯亮表示故障），通过本地维护终端（LMT）尝试连接 BBU（若能连接，排除传输整体中断）。
2. 光模块故障排查与处理
步骤 1：光模块状态检查
关闭 BBU 对应光口电源（或通过网管去激活光口），佩戴防静电手环，拔出光模块：检查外观（无破损、插针无弯曲）、标签（型号、波长与原配置一致）；
用光纤清洁笔清洁模块光口（避免指纹 / 灰尘污染），观察模块指示灯（如 SFP + 模块 “TX/RX” 灯，正常亮绿灯，灭表示无收发）。
步骤 2：光功率测试验证
用光功率计测试光模块发送功率（TX）：对比设备手册（如 10G 模块正常 - 8~-3dBm），若≤-10dBm 判定为发送故障；
测试接收功率（RX）：若≤-30dBm（或低于设备阈值），排除尾纤问题后，判定为接收故障。
步骤 3：更换光模块
佩戴防静电手环，插入备用光模块（听到 “咔” 声表示到位），紧固尾纤接口；
激活 BBU 光口，观察模块指示灯（TX/RX 灯亮绿灯），用 LMT 确认 “光口链路 UP”。
3. 尾纤故障排查与处理
步骤 1：尾纤物理检查
检查尾纤路由：是否过度弯曲（弯曲半径＜30mm）、挤压、破损（外皮开裂），接口是否松动（用力轻拽无位移）；
清洁尾纤接口：用酒精棉擦拭 LC/FC 接头（顺时针旋转擦拭），避免残留纤维。
步骤 2：光衰测试
用 OTDR 测试尾纤衰减值：单模尾纤正常衰减≤0.3dB/km（1310nm 波长），若某段衰减＞0.5dB/km，判定为尾纤断裂或接头故障；
分段测试：断开 BBU 与传输设备间的尾纤，分别测试 “BBU 侧尾纤”“传输侧尾纤”，定位故障段。
步骤 3：更换或修复尾纤
更换故障尾纤：按原路由布放新尾纤（避免与强电电缆并行），两端接口做好标签（如 “BBU 光口 0→PTN 端口 1”）；
熔接修复：若为尾纤接头故障，用熔接机重新熔接（熔接损耗≤0.1dB），热缩管保护后固定。
4. 传输故障排查与处理（协同传输专业）
步骤 1：传输设备状态检查
查看传输设备（如 PTN）对应端口指示灯：LINK 灯灭表示物理断链，ACT 灯闪烁异常表示链路不稳定；
登录传输设备网管：查询端口光功率（RX 应在 - 8~-12dBm）、误码率（正常为 0）、配置数据（VLAN、IP 地址是否与 BBU 匹配）。
步骤 2：传输链路测试
测试传输端口收发光功率：若收光功率过高（如＞-8dBm），加装光衰减器（衰减值 5~10dB）；若过低，排查上游传输设备或光缆；
误码测试：用传输测试仪在 BBU 与传输设备间打环，测试 15 分钟误码率（应无误码），若有误码，检查传输配置（如 MTU 值、封装协议）。
步骤 3：传输故障修复
传输端口光模块故障：更换同型号模块（操作同 BBU 光模块）；
配置错误：与传输代维协同，重新配置 VLAN、路由（确保 BBU的IP与传输网关互通）；
光缆故障：通知光缆维护团队抢修，期间可临时跳接备用光缆。
5. 恢复验证
BBU 通信验证：
网管侧：确认 “BBU 脱管” 告警清除，网元状态为 “正常”，光口链路 “UP”；
业务验证：拨打测试电话（语音接通率 100%）、测速（下载速率与配置值偏差≤10%），查看小区 RRC 连接成功率（≥99%）。
传输指标验证：传输链路误码率 = 0，时延≤50ms，光功率稳定在正常范围（-8~-12dBm）"

## 闭环管理	

1. 现场收尾
（1）整理尾纤：用扎带固定（避免受力），盘绕半径≥300mm，标签清晰可见；
（2）回收故障件：光模块 / 尾纤贴故障标签（如 “尾纤衰减超标”“光模块无发送”），单独存放于防静电袋；
（3）与接口人确认 “BBU 已恢复，业务正常”，签署《维护确认单》。
2. 工单闭环与台账更新
登录掌上运维 APP，进入代维掌中宝 - 故障工单，回复内容需包含：“XX 站点 BBU 因 XX（光模块 / 尾纤 / 传输）故障脱管，已更换 / 修复，当前网元在线，业务正常，告警清除”。
3. 复盘与改进
对重复故障（30 天内≥2 次）：
（1）光模块 / 尾纤：分析是否因环境恶劣（如湿度高、灰尘多），制定清洁周期（每月 1 次）；
（2）传输故障：优化传输路由（如增加备用链路），提升链路冗余度。"
